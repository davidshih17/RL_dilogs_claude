# Dilogarithm Simplification via Self-Supervised Oracle Trajectories

Code and data for reproducing the dilogarithm simplification results from:

**"Learning to Unscramble: Simplifying Symbolic Expressions via Self-Supervised Oracle Trajectories"**
David Shih
[arXiv:2603.11164](https://arxiv.org/abs/2603.11164)

## Overview

This repository implements a self-supervised learning approach for simplifying dilogarithm identities. The method formulates simplification as a Markov decision process (MDP), where oracle trajectories are generated by scrambling simple expressions and reversing the operations. A transformer-based policy network is trained on these trajectories to predict, at each step, which identity to apply to which term.

**Key result:** 99.9% solve rate (4,994/5,000) on a test set of dilogarithm expressions with varying target complexity and scramble depth.

## Quick Start: Evaluate Pre-Trained Model

```bash
pip install -r requirements.txt

# Evaluate the pre-trained model on the test set
python src/eval_sft_anticycle_harder.py \
    --checkpoint models/sft_harder_100k_v1/best_model.pt \
    --test_data data/harder_test_5k_oracle.pkl
```

## Full Reproduction

### 1. Generate Training Data

Training data consists of oracle trajectories: scrambled dilogarithm expressions paired with the sequence of identity applications that simplify them.

```bash
# Generate oracle training data (100k samples, parallelizable)
python scripts/generate_harder_with_oracle.py \
    --num_samples 1000 \
    --output_dir data/generated_oracle/

# For parallel generation (e.g., 100 workers x 1000 samples):
python scripts/generate_harder_oracle_worker.py \
    --worker_id 0 --num_samples 1000 --output_dir data/generated_chunks/

# Merge parallel outputs
python scripts/merge_harder_oracle_chunks.py \
    --input_dir data/generated_chunks/ \
    --output data/generated_train.pkl
```

Pre-generated training data is included at `data/harder_train_100k_oracle.pkl` (70MB, 100k samples).

### 2. Train Model

```bash
python src/train_sft_transformer.py \
    --oracle_dir data/harder_train_100k_oracle \
    --output_dir models/my_model \
    --epochs 200 \
    --batch_size 256 \
    --lr 1e-4
```

Note: `--oracle_dir` can point to either a directory of `.pkl` chunks or a single `.pkl` file.

### 3. Evaluate Model

```bash
python src/eval_sft_anticycle_harder.py \
    --checkpoint models/my_model/best_model.pt \
    --test_data data/harder_test_5k_oracle.pkl
```

### 4. Generate Paper Tables and Figures

```bash
python scripts/dilog_paper_table_and_figure_v3.py
```

This produces:
- `figures/dilog_solve_rate_vs_depth.pdf` — solve rate vs. scramble depth
- `figures/dilog_solve_rate_by_ns.pdf` — solve rate by target number of terms
- LaTeX-formatted comparison tables (printed to stdout)

## DSZ Baseline Comparison

Pre-computed evaluation results from the DSZ baseline (Dersy, Schwartz, Zhang, [arXiv:2206.04115](https://arxiv.org/abs/2206.04115)) are included at `data/dsz_baseline/eval.func_simple.test.0`. These are used by the paper plotting script for comparison.

To run the DSZ baseline yourself:
1. Clone the DSZ repository: https://github.com/aureliendersy/ML_Polylogarithms
2. Download their pre-trained `canonical_func` model
3. Evaluate on our test set (`data/paper_transformer_test_full.pkl`)

## Repository Structure

```
src/
    dilog_utils.py                 # DilogExpression class and identity operations
    dilog_env.py                   # Base Gymnasium environment
    dilog_env_equivariant.py       # Equivariant environment (3N action space)
    dilog_env_rllib.py             # Environment wrapper used by evaluation
    transformer_rl_module.py       # TransformerEncoder and TransformerPolicySFT models
    train_sft_transformer.py       # Training script
    eval_sft_anticycle_harder.py   # Evaluation with anti-cycle inference

scripts/
    generate_harder_with_oracle.py          # Generate oracle training data
    generate_harder_oracle_worker.py        # Parallel worker for data generation
    merge_harder_oracle_chunks.py           # Merge parallel data chunks
    get_overlap_indices.py                  # Detect train/test overlap
    convert_paper_data.py                   # Convert DSZ raw data format
    dilog_paper_table_and_figure_v3.py      # Generate paper tables and figures

data/
    harder_train_100k_oracle.pkl            # Training data (100k oracle trajectories)
    harder_test_5k_oracle.pkl               # Test data (5k samples)
    paper_transformer_test_full.pkl         # Full test set (5k, DSZ format)
    paper_test_set.pkl                      # Original DSZ test set (1.4k)
    overlap_indices_164.pkl                 # Train/test overlap indices
    dsz_baseline/eval.func_simple.test.0    # Pre-computed DSZ baseline results

models/
    sft_harder_100k_v1/best_model.pt        # Pre-trained model checkpoint
```
